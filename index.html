Modelos de Churn PF

No contexto de pessoa física, no âmbito bancário, os três principais problemas relacionados à perda de clientes são:

  • Inadimplência.
  • Churn voluntário, isto é, a solicitação de encerramento da conta.
  • Churn silencioso, ou seja, quando o cliente deixa de utilizar a conta bancária.

    • No nosso caso, não é diferente. Assim, foi identificado um alto volume de churn nessas três frentes mencionadas. Sendo assim, o nosso problema de negócio é identificar, de forma antecipada, os clientes com maior probabilidade de inadimplência ou churn voluntário. Como proposta de solução, foram sugeridos e desenvolvidos dois modelos preditivos para identificar os clientes com maior probabilidade de churn no próximo mês, de modo que os tomadores de decisão tenham um mês completo para atuar sobre esses clientes, reduzindo a taxa de churn e aumentando a taxa de retenção.

    Para o caso de inadimplência, utilizamos uma Decision Tree com as seguintes performances:

  • AUC: 89%
  • Precisão: 82%
  • Recall: 85%
  • Curva de ganho: em uma lista ordenada segundo as marcações do modelo, nos primeiros 50% dos clientes estão 75% dos clientes identificados como risco de churn.

    • Para o caso da evasão voluntária, utilizamos uma Regressão Logística com as seguintes performances:

  • AUC: 85%
  • Precisão: 87%
  • Recall: 79%
  • Curva de ganho: em uma lista ordenada segundo as marcações do modelo, nos primeiros 50% dos clientes estão 81% dos clientes identificados como risco de churn.

    • Para que as ações sejam tomadas de forma mais efetiva e ágil, propus a construção de um dashboard atualizado mensalmente de forma automatizada, diretamente integrado aos modelos de churn, com filtros interativos para que o tomador de decisão possa investigar e elaborar planos de ação conforme os grupos e seus respectivos níveis de risco. Além disso, construímos 5 categorias de risco para facilitar a gestão. As categorias foram definidas da seguinte forma:

  • Risco Altíssimo: probabilidade acima de 80% de churn.
  • Risco Alto: probabilidade entre 80% e 60% de churn.
  • Risco Moderado: probabilidade entre 59% e 40% de churn.
  • Risco Baixo: probabilidade entre 39% e 20% de churn.
  • Risco Baixíssimo: probabilidade abaixo de 19% de churn.

    • Ferramentas utilizadas: SAS Viya, SAS Visual Analytics, Python e AWS S3.

    Modelo de Propensão de Aceitar Parcelados

    Ao entender como o operacional de visitas dos gerentes comerciais era realizado para a prospecção de clientes, foi identificada a oportunidade de implementar uma solução para um problema de negócio: estimar a probabilidade de um cliente aceitar um parcelado. A solução proposta foi a definição de uma ordem de visitas, com o objetivo de priorizar clientes com maior propensão a contratar um parcelado. Conseguimos implementar essa solução com sucesso por meio de um modelo de regressão logística, que apresentou uma recuperação de 73%, precisão de 80% e uma curva de ganho indicando que, nos primeiros 50% da lista ordenada pela probabilidade estimada pelo modelo, estavam concentrados 80% dos clientes interessados.

    Para a oferta desse produto, era utilizado uma ferramenta de disparo para canais de comunicação. Sem o modelo atuando, era necessário, aproximadamente, 18.000 disparos para converter 8. Com o modelo, foi necessário apenas 1.800 disparos para alcançar 5 conversões. Isto é, conseguimos ganhar uma performance com redução de 90%. Além disso, em termos financeiros, projetou-se um ganho mensal R$ 24.000 de margem de comtribuição. Para corroborar tudo isto, utilizamos teste A/B aplicando o teste de proporção (z-teste).

    Ferramentas Utilizadas: SAS Viya, SAS Visual Analytics, Python e AWS S3.

    Recomendação de Produtos - Verticalização

    Desenvolvemos um sistema de recomendação voltado para clientes que já possuem ao menos um produto no banco, com foco na verticalização do relacionamento. Devido à complexidade do sistema bancário, selecionamos clientes sem comprometimento de crédito, buscando garantir consistência nas sugestões. Para tornar as recomendações mais precisas, coletamos diversos atributos para uma comparação justa. Usamos informações cadastrais — como estado, faturamento e dívida de mercado — e dados bancários, como número de parcelamentos de empréstimos, quantidade de seguros e outras variáveis relevantes. A abordagem técnica incluiu um pré-processamento personalizado, visando uma vetorização eficiente. Com isso, aplicamos dois métodos de recomendação:

  • Filtragem colaborativa.
  • Filtragem por similaridade do cosseno.

    • Após a construção do modelo, implementamos uma camada de refinamento de produtos. Por exemplo, não faz sentido recomendar empréstimos a clientes com bloqueio de proposta. Esse filtro garante uma recomendação precisa e alinhada ao perfil do cliente. O produto final inclui:

  • Um dashboard operacional.
  • Integrações com canais digitais para construção de carrosséis inteligentes.
  • Ferramentas para gerentes comerciais fazerem ofertas mais estratégicas.

    • O sistema de recomendação agregou ótimos resultados financeiros. Para avaliar esses resultados, realizamos um teste A/B. No grupo controle, analisando todos os produtos, tivemos uma conversão de 22% com um ganho de receita de R$ 59.351,00 em público de aproximadamente 19.000 clientes. Em contrapartida, com o nosso grupo teste, obtivemos uma conversão de 39% com um ganho de receita de R$ 338.431,00 atuando em público de aproximadamente 8.500 clientes. O sistema de recomendação é, estatisticamente efetivo, trazendo ganhos significativos, melhorando a personalização de ofertas de produtos e, utilizando menos esforço.

    Ferramentas Utilizadas: SAS Viya, SAS Visual Analytics, Python e AWS S3.

    Modelo de Churn de Clientes

    Historicamente, o banco compreendia o abandono de produtos específicos pelos clientes, mas não havia uma visão clara sobre a saída total do cliente do ecossistema bancário. Para suprir essa lacuna, desenvolvemos — em parceria com outros cientistas de dados — um sistema baseado em machine learning para prever o churn no mês seguinte.

    Identificamos os sete indicadores mais estratégicos para o nosso contexto bancário, analisando suas tendências de alta ou queda. A partir disso, construímos sete modelos preditivos, cuja combinação gera uma pontuação final indicando a probabilidade de churn. Na parte técnica, utilizamos o algoritmo Gradient Boosting Classifier, que alcançou resultados expressivos após refinamentos:

  • Precisão Geral: 85%.
  • Recall Médio: 88%.
  • AUC: 90%.

    • Essas previsões possibilitaram ações direcionadas para retenção de clientes. O projeto resultou em:

  • Um dashboard operacional para acompanhamento dos clientes com alto risco de churn.
  • Suporte para decisão estratégica por superintendentes, gerentes regionais e comerciais.

    • Ferramentas Utilizadas: SAS Viya, SAS Visual Analytics, Python e AWS S3.

    Inteligência de Dados - Ouvidoria

    Com o objetivo de potencializar as ações da área de Ouvidoria utilizando dados, construi, a pedido do gestor, indicadores-chaves para os principais canais de reclamação, que no caso são, BACEN, Consumidor.gov e Canais Regulados(Interno). Nesse situação, foi desenvolvido indicadores como volumetria de reclamações, os 5 maiores motivos de reclamações, distribuição de reclamações por faixa de horário para melhor alocamento de força de trabalho entre outros indicadores. Com esse dashboard e a geração de insights e inteligência, foram gerados 8 ações para a melhoria no tratamento dos casos. São eles:

  • Cancelamento de Cartão
  • Cancelamento de Conta
  • Cobrança de Dívida Prescrita
  • Ligação Indevida de Não-Cliente
  • Operações de Crédito
  • SCR - Registro de Dívidas no BACEN

    • Essas ações geraram bons resultados. Comparando o 1º Semestre de 2024(sem ações) com o 1º Semestre de 2025(com ações) temos os seguintes resultados:

  • Redução de 100% no Golpe de Operações de Cartão
  • Redução de 76% no Cancelamento de Conta Corrente
  • Redução de 56% nos Débitos com o Tribanco
  • Redução de 100% na Reclamação de Terceiros
  • Redução de 9% em SCR

    • Ferramentas Utilizadas: SAS Viya, SAS Visual Analytics e AWS S3.
