Modelo de Propensão de Aceitar Parcelados

Ao entender como o operacional de visitas dos gerentes comerciais era realizado para a prospecção de clientes, foi identificada a oportunidade de implementar uma solução para um problema de negócio: estimar a probabilidade de um cliente aceitar um parcelado. A solução proposta foi a definição de uma ordem de visitas, com o objetivo de priorizar clientes com maior propensão a contratar um parcelado. Conseguimos implementar essa solução com sucesso por meio de um modelo de regressão logística, que apresentou uma recuperação de 73%, precisão de 80% e uma curva de ganho indicando que, nos primeiros 50% da lista ordenada pela probabilidade estimada pelo modelo, estavam concentrados 80% dos clientes interessados.

Para a oferta desse produto, era utilizado uma ferramenta de disparo para canais de comunicação. Sem o modelo atuando, era necessário, aproximadamente, 18.000 disparos para converter 8. Com o modelo, foi necessário apenas 1.800 disparos para alcançar 5 conversões. Isto é, conseguimos ganhar uma performance com redução de 90%. Para corroborar isto, utilizamos teste A/B aplicando o teste de proporção (z-teste).

Ferramentas Utilizadas: SAS Viya, SAS Visual Analytics, Python e AWS S3.

Recomendação de Produtos - Verticalização

Desenvolvemos um sistema de recomendação voltado para clientes que já possuem ao menos um produto no banco, com foco na verticalização do relacionamento. Devido à complexidade do sistema bancário, selecionamos clientes sem comprometimento de crédito, buscando garantir consistência nas sugestões. Para tornar as recomendações mais precisas, coletamos diversos atributos para uma comparação justa. Usamos informações cadastrais — como estado, faturamento e dívida de mercado — e dados bancários, como número de parcelamentos de empréstimos, quantidade de seguros e outras variáveis relevantes. A abordagem técnica incluiu um pré-processamento personalizado, visando uma vetorização eficiente. Com isso, aplicamos dois métodos de recomendação:

  • Filtragem colaborativa.
  • Filtragem por similaridade do cosseno.

    • Após a construção do modelo, implementamos uma camada de refinamento de produtos. Por exemplo, não faz sentido recomendar empréstimos a clientes com bloqueio de proposta. Esse filtro garante uma recomendação precisa e alinhada ao perfil do cliente. O produto final inclui:

  • Um dashboard operacional.
  • Integrações com canais digitais para construção de carrosséis inteligentes.
  • Ferramentas para gerentes comerciais fazerem ofertas mais estratégicas.

    • Ferramentas Utilizadas: SAS Viya, SAS Visual Analytics, Python e AWS S3.

    Modelo de Churn de Clientes

    Historicamente, o banco compreendia o abandono de produtos específicos pelos clientes, mas não havia uma visão clara sobre a saída total do cliente do ecossistema bancário. Para suprir essa lacuna, desenvolvemos — em parceria com outros cientistas de dados — um sistema baseado em machine learning para prever o churn no mês seguinte.

    Identificamos os sete indicadores mais estratégicos para o nosso contexto bancário, analisando suas tendências de alta ou queda. A partir disso, construímos sete modelos preditivos, cuja combinação gera uma pontuação final indicando a probabilidade de churn. Na parte técnica, utilizamos o algoritmo Gradient Boosting Classifier, que alcançou resultados expressivos após refinamentos:

  • Precisão Geral: 85%.
  • Recall Médio: 88%.
  • AUC: 90%.

    • Essas previsões possibilitaram ações direcionadas para retenção de clientes. O projeto resultou em:

  • Um dashboard operacional para acompanhamento dos clientes com alto risco de churn.
  • Suporte para decisão estratégica por superintendentes, gerentes regionais e comerciais.

    • Ferramentas Utilizadas: SAS Viya, SAS Visual Analytics, Python e AWS S3.

    Inteligência de Dados - Ouvidoria

    Com o objetivo de potencializar as ações da área de Ouvidoria utilizando dados, construi, a pedido do gestor, indicadores-chaves para os principais canais de reclamação, que no caso são, BACEN, Consumidor.gov e Canais Regulados(Interno). Nesse situação, foi desenvolvido indicadores como volumetria de reclamações, os 5 maiores motivos de reclamações, distribuição de reclamações por faixa de horário para melhor alocamento de força de trabalho entre outros indicadores.

    Ferramentas Utilizadas: SAS Viya, SAS Visual Analytics e AWS S3.
